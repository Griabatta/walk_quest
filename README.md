# Вечерний квест

Личный PWA-квест для вечерних прогулок на Android и iPhone.

## Что умеет

- определяет текущую геолокацию;
- принимает цель в шагах;
- подбирает несколько случайных конечных точек и проверяет для них реальные пешие маршруты через OSRM/OpenStreetMap;
- выбирает маршрут, который ближе всего к заданному количеству шагов;
- показывает цель и линию маршрута на карте OpenStreetMap;
- даёт ссылку на пеший маршрут в Google Maps;
- отслеживает расстояние до цели;
- засчитывает победу в радиусе 50 / 75 / 100 м;
- включает звук, вибрацию и конфетти;
- сохраняет историю прогулок в localStorage;
- устанавливается на главный экран Android и iPhone как PWA.

## Как запустить на Android

Для геолокации нужен HTTPS. Самый простой вариант без Google Play:

1. Загрузи папку проекта на GitHub Pages, Netlify Drop, Vercel или Cloudflare Pages.
2. Открой ссылку на телефоне в Chrome.
3. Разреши доступ к геолокации.
4. В меню Chrome нажми «Добавить на главный экран» или «Установить приложение».


## Как запустить на iPhone

Для iPhone тоже нужен HTTPS-сайт. Установка без App Store делается так:

1. Загрузи папку проекта на GitHub Pages, Netlify Drop, Vercel или Cloudflare Pages.
2. Открой ссылку именно в Safari на iPhone.
3. Нажми кнопку «Поделиться».
4. Выбери «На экран “Домой”» / «Add to Home Screen».
5. Открой приложение с иконки на домашнем экране и разреши геолокацию.

На iPhone приложение лучше держать открытым во время прогулки. В фоне iOS может останавливать или сильно ограничивать обновление геолокации у веб-приложений. Вибрация через Vibration API в Safari обычно не работает, поэтому финальный сигнал на iPhone — это звук, экран победы и конфетти.

## Ограничения этой версии

- Приложение не считает реальные шаги. Оно переводит заданные шаги в примерную длину пешего маршрута.
- Пешие маршруты считаются через публичный сервер `routing.openstreetmap.de/routed-foot`, поэтому нужен интернет. Если сервер временно недоступен, приложение выберет цель запасным способом по примерной прямой дистанции.
- Публичный сервер подходит для личного использования и тестов, но не для большой нагрузки. Для полноценного продукта лучше подключить свой routing API или свой OSRM-сервер.
- Google Maps может предложить немного другой маршрут, потому что пересчитывает путь своим алгоритмом.
- В фоне Android/Chrome и iOS/Safari могут реже обновлять геолокацию, поэтому лучше держать приложение открытым во время прогулки.
- На iPhone вибрация может не сработать: Safari не поддерживает обычный `navigator.vibrate()` так же, как Android-браузеры.

## Настройки

В приложении можно менять:

- количество шагов;
- примерную длину шага;
- радиус победы.

## Файлы

- `index.html` — интерфейс;
- `styles.css` — оформление;
- `app.js` — логика квеста;
- `manifest.webmanifest` — установка как PWA;
- `service-worker.js` — базовый офлайн-кеш;
- `icons/` — иконки приложения.


## Как теперь выбирается цель

1. Приложение переводит шаги в метры по длине шага.
2. Генерирует несколько случайных точек вокруг текущего местоположения.
3. Для каждой точки запрашивает пеший маршрут.
4. Сравнивает длину маршрута с твоей целью.
5. Берёт самый близкий вариант и рисует его на карте.
